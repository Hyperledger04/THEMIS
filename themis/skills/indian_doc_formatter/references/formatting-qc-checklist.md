# Formatting QC Checklist

Run this checklist before delivering any formatted document. Tag issues in the
change log — do not silently fix substantive content.

## 1. Numbering Integrity
- [ ] All clause numbers are sequential with no gaps or duplicates
- [ ] Multi-level numbering is consistent (1 → 1.1 → 1.1.1 or 1 → (a) → (i))
- [ ] Schedule numbering restarts correctly within each Schedule
- [ ] Annexure labels match body references (e.g., "Schedule 1" in body ↔
      "SCHEDULE 1" heading)

## 2. Cross-Reference Accuracy
- [ ] All "Clause X.Y" references point to an existing clause number
- [ ] No orphaned cross-references (references to deleted/renumbered clauses)
- [ ] "Schedule X" and "Annexure X" references resolve

## 3. Defined-Term Consistency
- [ ] Every defined term appears in the Definitions clause (or is defined
      inline on first use)
- [ ] All defined terms use consistent capitalisation throughout the document
- [ ] No term is used before it is defined (flag; do not invent a definition)
- [ ] Defined terms in signature block match defined terms in body

## 4. TOC Accuracy (if TOC present)
- [ ] TOC entries match actual heading text exactly
- [ ] TOC page numbers are accurate (or TOC is set to auto-update on open)

## 5. Signature Block Completeness
- [ ] Number of signature blocks matches number of parties named in recitals
- [ ] Each block has: Name, Designation, Date, Place fields
- [ ] Witness lines present if required by document type

## 6. Style Consistency
- [ ] No paragraphs use inline font overrides that conflict with the style profile
- [ ] No mixed indentation (some paragraphs indented, others not)
- [ ] Spacing is uniform throughout (no extra blank lines between clauses)

## 7. Legal Compliance Flags
- [ ] Change log present with [REQUIRES ATTORNEY REVIEW] note at top
- [ ] All [REQUIRES VERIFICATION] tags have been either resolved or left for
      attorney review
- [ ] No substantive content added or removed (compare paragraph count to
      source if possible)
